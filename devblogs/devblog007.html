Teslov's Secret Notes


Weapon System 2.0


For long time weapons were nothing more than meshes attached to player models. If weapon needed some additional data ( like special abilities ) it was handled by another object that spawned on runtime. It was of course terrible and hard to manage. I was probably drunk when I thought of it. Anyway, it got reworked and all the data about your weapon is stored in your weapon. This allows us to make unique items a lot more easily.

Building System


Building system is currently pretty simple. Grab a hammer, element template will show up. Press right click to change selected piece or press left click to build it. Every part cost 100 wood for now, it'll of course change in the future.
Fun Fact: Building was implemented before inventory API was a thing so it was impossible to manipulate it easily ( it is now, thankfully )

Magic System


This one is my favourited. Magic is handled like weapons but exists only temporary as it's not stored in inventory. It is instead stored in array of available spells. This way you can learn spells or you can cast them with another item like scroll or rune.
There's also spellbook widget where you can browse your spells.

Consumables


Another thing added, items you can use from your inventory. It includes things like bread, water, potions or magical scrolls. System is very extendable and can be easily used to make any kind of item as every item can have it's own script.

Containers


Inventory system got moved to external component and got it's own API. Now it's a lot easier to manipulate backpack so I've added containers.
Lootable things can now be plundered with special quick-loot interface a little bit based on certain FPS game with RPG elements that took place in wastelands.

Animation Tweaks


Items now have Type which dictates what animation does it use. All anims are now nicely blended and show on both first and third person

Factions


Factions allow AI to understand who's an enemy and who's a friend. Faction include things like "undead" or "predator" and allow for things like making Wolves attack Deer. System is of course runtime-editable and includes players as they can also join one of four major factions.

Harvestables Rework


For testing purposes, at the beginning of the project we used simple actors for gatherable resources. This way was pretty bad because it's both performance unfriendly and make map-making a very boring task.
I've rewrote the system to use foliage with custom instanced mesh solution that allows for replication.
( Foliage is not able to be replicated in UE4 by default )
